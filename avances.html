Nombre del proyecto

Instituto Eureka

Actividad principal de la institución

El Instituto "Eureka" se especializa en ofrecer clases personalizadas para estudiantes universitarios que buscan apoyo académico. Su enfoque está en brindar métodos de estudio flexibles y accesibles.

Objetivos del sitio web

El sitio web tiene como objetivo presentar los servicios del instituto, permitir la reserva de clases de manera intuitiva (dejando elegir al profesor, el horario y curso), proporcionar acceso a una biblioteca digital de recursos académicos, ofrecer un foro para la interacción entre estudiantes, como así también, una pestaña de preguntas frecuentes para que los usuarios despejen todas sus dudas. Por ultimo, se detallarán los métodos de pago y contacto con la institución.

Concepto general del sitio web

El sitio web tiene un concepto centrado en la accesibilidad y flexibilidad, diseñado para ser una herramienta práctica y útil para estudiantes universitarios. Ofrece servicios que facilitan la mejora del rendimiento académico.

Estrategias para atraer tráfico

Para atraer tráfico al sitio web, se implementarán estrategias/campañas publicitarias en redes sociales enfocadas en estudiantes universitarios, creación de contenido educativo en un blog, testimonios de estudiantes satisfechos y colaboraciones con influencers del ámbito académico.

Público Objetivo

El sitio web está destinado a estudiantes universitarios, sin distinción de género. Estos estudiantes tienen aficiones relacionadas con el estudio eficiente, el uso de herramientas tecnológicas para el aprendizaje y la superación académica. Su nivel económico es medio, y buscan soluciones accesibles para mejorar su rendimiento académico y entender los temas que estudian. Los intereses principales de este grupo incluyen la personalización del aprendizaje, el acceso a recursos educativos gratuitos/alcanzables, y la interacción con compañeros y tutores.

El sitio se adapta a sus necesidades ofreciendo una plataforma fácil de usar, responsiva desde cualquier dispositivo y navegador. Cuenta con horarios flexibles para la reserva de clases, recursos de estudio en línea, foros de discusión interactivos y métodos de pago seguros, todo orientado a facilitar su experiencia de aprendizaje y resolver sus desafíos académicos.

Buenas Prácticas de Accesibilidad

Las estrategias que se utilizan para que la página sea accesible son:

  1. 1. Proporcionar alternativas textuales (Perceptible): Confirmar que todo contenido no textual tenga una alternativa textual para que los usuarios puedan comprenderlo con lectores de pantalla o tecnologías asistivas.
  2. 2. Proporcionar suficiente contraste de colores (Comprensible): Asegurar de que el texto sea legible y distinguible del fondo, especialmente para personas con discapacidad visual o daltonismo.
  3. 3. Crear contenido adaptable (Perceptible): El contenido debe ajustarse a diferentes dispositivos y preferencias del usuario, como tamaños de pantalla o modos de visualización. Por eso, se utilizan media queries.

Buenas Prácticas de Usabilidad

Las 5 heurísticas de usabilidad de Jakob Nielsen que se pueden visualizar en el sitio son:

  • Visibilidad del estado del sistema

    • El sistema debe mantener informado al usuario sobre lo que esta ocurriendo en la web a cada momento. Cada palabra explica que es cada cosa.

  • Consistencia entre el sistema y el mundo real

    • El sistema tiene que hablar el idioma del usuario. Se debe utilizar palabras, frases y conceptos que le sean familiares. De esta forma, se visualiza que el contenido de la página está en español

  • Consistencia y estándares

    • Los usuarios no deben verse obligados si diferentes palabras, situaciones o acciones significan lo mismo. Se debe seguir las convenciones establecidas en la plataforma y la industria. En el sitio, el menú hamburguesa funciona perfecto.

  • Prevención de errores

    • Los buenos mensajes de error son importantes. Hay que crear un diseño que evite que los errores ocurran. El form de reservas es un gran ejemplo, ya que muestra un error si este no está completo.

  • Ayuda y documentación:

    • Lo ideal es que un sistema no necesite ninguna explicación adicional. Pero en algunos casos, se debe proporcionar una documentación para ayudar a los usuarios a entender cómo completar sus tareas. La sección de FAQ (Preguntas frecuentes) representa este principio.
